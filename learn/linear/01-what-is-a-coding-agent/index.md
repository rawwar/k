# What Is a Coding Agent?

The emergence of AI coding agents represents a fundamental shift in how software is written. Unlike simple code completion tools or chatbots that respond to isolated questions, coding agents operate autonomously within development environments — reading files, executing commands, running tests, and iterating on solutions. This chapter establishes the foundational understanding of what coding agents are, how they differ from earlier AI-assisted development tools, and why they matter.

We begin by surveying the current landscape of production coding agents including Claude Code, OpenCode, Pi, and Codex. By studying how each system approaches the problem of autonomous code generation and manipulation, you will develop an intuition for the common architectural patterns that underlie all of them. This comparative analysis reveals that despite surface-level differences, these tools share a remarkably consistent core design.

By the end of this chapter, you will have a clear mental model of what a coding agent is, how the major players differ, and what we will build throughout the rest of this tutorial track. You will also understand why Rust is our language of choice and what the final system will look like.

## Learning Objectives
- Define what a coding agent is and how it differs from chatbots and code completion tools
- Identify the major production coding agents and their distinguishing characteristics
- Recognize the common architectural patterns shared across all coding agents
- Understand the motivation for building a custom coding agent from scratch
- Articulate the high-level architecture of the system we will build in this track
